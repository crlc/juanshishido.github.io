General Linear Models

Simple Linear Regression

A line can be described by the equation \(y = mx + b\), where \(b\) is the \(y\)-intercept and \(m\) is the slope. Simple linear regression uses this idea to describe the relationship between two variables—one dependent, the \(y\), and one explanatory, the \(x\). The process uses the data to find the line that "best" fits it.

Because each observation has its own \(x\) and \(y\) value, we use subscripts to denote them. In addition, unless the data form a single straight line, we will need to introduce an additional concept—errors.

Relative to the true line that describes \(x\) and \(y\), the observed data will be distributed both above and below it. The differences between these points and the line is referred to as the "error." To capture this, we add a new term to our equation. \[y_i = \beta_0 + \beta_1 x_i + \epsilon_i\] In this form, in which the terms have been slightly reordered from the equation above, \(\beta_0\) represents the \(y\)-intercept, \(\beta_1\) represents the slope, and \(\epsilon_i\) represents the error. Notice that the \(y\), \(x\), and \(\epsilon\) terms all have subscripts, but the \(\beta\)s do not. This implies that there is a single \(y\)-intercept and slope—the true line.

While we have specified the relationship we believe the data have, the model is not fully specified. For this, we must be explicit about our assumptions of \(\epsilon\). We can, for example, assume that \(\epsilon\) are independent and identically distributed with a zero mean and \(\sigma^2\) variance.

Together, the equation and the assumptions about \(\epsilon\) describe the model.

Matrix Form

It is useful to specify our model in matrix form. We can think of \(y_i\), \(x_i\), and \(\epsilon_i\) as being vectors with \(n\) values, where \(i \in 1,\ldots,n\). \(y_i\), for example, can be written as \(\vec{y}\). Our equation becomes:

\[\vec{y} = \beta_0 + \beta_1 \vec{x} + \vec{\epsilon}\]

Using this notation, the \(\beta\) values are scalars. However, with what we've described thus far, \(\beta_0\) doesn't have a vector of values to be multiplied with. So, we add a vector of ones of length \(n\).

Our system of \(n\) equations can be represented as the following matrix addition equation.

\begin{split} \left[\begin{matrix}y_1\\ \vdots\\ y_n\end{matrix}\right] = \beta_0 \left[\begin{matrix}1\\ \vdots\\ 1\end{matrix}\right] + \beta_1 \left[\begin{matrix}x_1\\ \vdots\\ x_n\end{matrix}\right] + \left[\begin{matrix}\epsilon_1\\ \vdots\\ \epsilon_n\end{matrix}\right] \end{split}

We can restate this in terms of matrix multiplication.

\begin{split} \left[\begin{matrix}y_1\\ \vdots\\ y_n\end{matrix}\right] = \left[\begin{matrix}1 & x_1\\ \vdots & \vdots\\ 1 & x_n\end{matrix}\right] \left[\begin{matrix}\beta_0\\ \beta_1\end{matrix}\right] + \left[\begin{matrix}\epsilon_1\\ \vdots\\ \epsilon_n\end{matrix}\right] \end{split}

Here, we've simply combined the vector of ones and \(x_i\) into a matrix, which we refer to as \(X\), called the design matrix, and combined the \(\beta\)s into a column vector, which we refer to as \(\vec{\beta}\), called the parameter vector. In this example, \(X\) is an \(n \times p\) matrix and \(\vec{\beta}\) can be thought of as a \(p \times 1\) matrix. Because the number of columns in \(X\) equals the number of rows in \(\vec{\beta}\), they can be multiplied.

This can be rewritten as follows.

\[\vec{y} = X\vec{\beta} + \vec{\epsilon} \]

Multiple Regression

Often, outcomes for our variable of interest, \(y_i\), depend on two or more variables. This is referred to as a multiple linear regression model, which, for the case of two variables, can be written as:

\[y_i = \beta_0 + \beta_1 x_{1i} + \beta_2 x_{2i} + \epsilon_i\]

This model, as any model with an arbitrary number of parameters, can still be expressed as:

\[\vec{y} = X\vec{\beta} + \vec{\epsilon} \]

For each parameter, \(p\), add a column to \(X\) and an element to \(\vec{\beta}\).
